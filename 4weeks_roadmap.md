## 1주차 — 기획 & 설계

### Day 1~2: 문제 정의 & 페르소나 스키마
- [ ] 마케터 페인포인트 3~5개로 압축 (예: "카피 여러 개 만드는 데 시간 소요", "타겟별 반응 예측이 감에 의존", "A/B 테스트 실제 집행 전 사전 검증 수단 부재")
- [ ] **가상 페르소나 축 설계** — 이게 프로젝트의 핵심 차별점이므로 신중하게:
  - 인구통계 축: 연령대, 성별, 소득 구간
  - 심리 축: 소비 성향(가성비형/프리미엄형/트렌드추종형), 광고 리터러시(광고에 냉소적/우호적)
  - 채널 맥락: 어느 매체에서 노출되는지 (인스타 릴스 vs 검색광고 vs 배너)
  - → 이 조합으로 최소 6~8개 페르소나 템플릿 정의 (예: "20대 가성비 중시 인스타 헤비유저", "40대 프리미엄 지향 검색광고 노출군" 등)
- [ ] 페르소나별 "평가 기준" 정의 — 단순 호감도가 아니라 클릭 의도, 신뢰도, 브랜드 적합성 등 다축 스코어로 설계 (인사이트 리포트의 근거가 됨)

### Day 3: KPI & 유저플로우
- [ ] 서비스 자체 KPI 정의 (예: 카피 생성→리포트 확인까지 소요 시간, 사용자가 "유용하다"고 느끼는 인사이트 비율)
- [ ] 유저플로우 다이어그램: 카피 초안 입력 → 타겟 세그먼트 선택 → 변형안 N개 생성 → 페르소나 패널 자동 평가 → 비교 리포트 출력
- [ ] 화면 와이어프레임 3~4장 (입력 화면, 변형안 리스트, 페르소나 평가 상세, 대시보드)

### Day 4~5: 아키텍처 & 데이터 모델
- [ ] 시스템 아키텍처 다이어그램: React ↔ FastAPI ↔ LLM API ↔ (선택) DB
- [ ] DB 스키마 초안 (campaigns, copy_variants, personas, evaluations 테이블 등) — Supabase or PostgreSQL 중 택1 (ScholarFit에서 Supabase 써봤으면 재사용해서 속도 낼 것)
- [ ] API 엔드포인트 목록 초안 (`/generate-variants`, `/evaluate`, `/report/{campaign_id}` 등)

### Day 6~7: AI 프롬프트 1차 설계 + 검증
- [ ] 카피 변형 생성 프롬프트 초안 작성 → 실제 API로 5~10개 샘플 테스트
- [ ] 페르소나 평가 프롬프트 초안 작성 → 같은 카피에 대해 페르소나별로 실제 차별화된 반응이 나오는지 검증 (다 비슷한 점수면 페르소나 설계를 다시 다듬어야 함 — 여기서 가장 많은 시행착오 예상)
- [ ] **주말 체크포인트**: 기획 문서 v1 완성 (PDF or Notion 링크), 프롬프트 검증 결과 정리

## 2주차 — 백엔드 & AI 파이프라인

### Day 8~9: FastAPI 기본 구조
- [ ] 프로젝트 스캐폴딩, 환경변수/시크릿 관리, CORS 설정
- [ ] DB 연결, 기본 CRUD (campaign 생성, copy 저장)
- [ ] `/generate-variants` 엔드포인트 — LLM API 호출, N개 변형안 생성 및 파싱

### Day 10~12: 페르소나 평가 엔진
- [ ] `/evaluate` 엔드포인트 — 카피 변형 × 페르소나 조합으로 배치 평가 호출 (비용/속도 고려해 병렬 처리 또는 배치 API 검토)
- [ ] 평가 결과 구조화 (JSON 스키마 강제 — 프롬프트에 "JSON만 출력" 명시, 파싱 실패 시 재시도 로직)
- [ ] **환각 방지 전략 적용** (AI 활용 보고서에 그대로 쓸 내용):
  - 평가 근거를 반드시 함께 출력하게 해서 점수만 나오는 게 아니라 "왜 이 점수인지" 확인 가능하게
  - 페르소나 설정을 매 호출마다 시스템 프롬프트에 고정 주입해 일관성 유지
  - 동일 입력 반복 테스트로 점수 변동성(variance) 체크 → 너무 들쭉날쭉하면 temperature 조정

### Day 13~14: 리포트 집계 로직
- [ ] `/report/{campaign_id}` — 페르소나별 점수를 집계해서 "어떤 카피가 어떤 타겟에 강한지" 매트릭스 형태로 반환
- [ ] 간단한 랭킹/추천 로직 (예: 페르소나 그룹별 최고 점수 카피 하이라이트)
- [ ] **주말 체크포인트**: Postman/Swagger로 전체 API 플로우 end-to-end 테스트 통과

## 리스크 포인트 미리 체크
1. **페르소나 평가의 신뢰성** — 가장 큰 리스크. 1주차에 미리 검증 안 하면 3주차에 대시보드 만들어놨는데 데이터가 밋밋해서 갈아엎을 수 있음
2. **API 비용** — N개 변형 × M개 페르소나 조합이면 호출 수가 빠르게 늘어남. 캐싱이나 배치 처리 필수
3. **일정 압박** — 방학 중 다른 일정(연구실, 팀플 등)과 겹칠 경우 2주차 AI 파이프라인 구간이 가장 밀리기 쉬움. 여기 버퍼 하루 정도 확보 권장
